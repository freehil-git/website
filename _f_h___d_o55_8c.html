FH_ErrorInfo FH_DO_Init_DO55 ( )

This function initializes the Digital Output 55
The Digital Output 55 could be any interested digital output of the micro-controller on which FH source code is ported
FH user should call the initialization function of the interested digital output here.

Parameters
None
Returns
FH_ErrorInfo is returned
Note
To have a clean code, just include the related header file
Then just call the function here
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${DO55}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_DO}    Command=${DO_Commands_Init}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  DO55 => It indicates the Digital Output 55
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_DO => It is the function in the message frame
  DO_Commands_Init => It is the command of the function in the message frame
Warning
To be able to set or reset the Digital Output 55 of the micro-controller on which FH source code is ported, this function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_DO_SetStatus_DO55 function from RobotFramework, this function shall be invoked in order to initialize the digital output

◆ FH_DO_SetStatus_DO55()

FH_ErrorInfo FH_DO_SetStatus_DO55 ( uint8_t Status)

This function sets or resets the Digital Output 55
The Digital Output 55 could be any interested digital output of the micro-controller on which FH source code is ported
FH user should call the set and reset functions of the interested digital output here.

Parameters
StatusIf Status is 1, the set function of the Digital Output 55 shall be called
If Status is 0, the reset function of the Digital Output 55 shall be called
Returns
FH_ErrorInfo is returned
Note
To have a clean code, just include the related header file
Then just call the function here
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${DO55}    ${DO_Set}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_DO}    Command=${DO_Commands_SetStatus}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  DO55 => It indicates the Digital Output 55
  DO_Set => It indicates the Digital Output 55 shall be Set (DO_Reset indicates the Digital Output 55 shall be Reset)
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_DO => It is the function in the message frame
  DO_Commands_SetStatus => It is the command of the function in the message frame
Warning
To be able to set or reset the Digital Output 55 of the micro-controller on which FH source code is ported, the FH_DO_Init_DO55 function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_DO_SetStatus_DO55 function from RobotFramework, FH_DO_Init_DO55 function shall be invoked in order to initialize the digital output
