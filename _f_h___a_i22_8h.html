◆ FH_AI_GetStatus_AI22()

FH_ErrorInfo FH_AI_GetStatus_AI22 ( FH_RFCommunication_Message * fh_RFCommunication_Message,
uint32_t * Data )

This function reads the Analog Input 22
The Analog Input 22 could be any interested Analog Input of the micro-controller on which FH source code is ported
FH user should call the read function of the interested Analog Input here.

Parameters
Data*Data will be set with the value of Analog Input 22
Returns
FH_ErrorInfo is returned
Note
To have a clean code, just include the related header file
Then just call the function here
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${AI22}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_AI}    Command=${AI_Commands_GetStatus}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  AI22 => It indicates the Analog Input 22
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_AI => It is the function in the message frame
  AI_Commands_GetStatus => It is the command of the function in the message frame
Warning
To be able to read the Analog Input 22 of the micro-controller on which FH source code is ported, the FH_AI_Init_AI22 function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_AI_GetStatus_AI22 function from RobotFramework, FH_AI_Init_AI22 function shall be invoked in order to initialize the Analog Input
