FH_ErrorInfo FH_CAN_GSD16_CAN02 ( FH_RFCommunication_Message * fh_RFCommunication_Message,
uint8_t * SharedDataLength,
uint8_t * SharedData )

This function gets the shared data buffer (FH_CAN02_SharedDataBuf_16Bits) of the CAN 02
The CAN 02 could be any interested CAN of the micro-controller on which FH source code is ported
FH_CAN02_SharedDataBuf_16Bits is a shared global 16 bit array which is a data bridge between RobotFramework and FH user code
FH_CAN02_SharedDataBuf_16Bits is optional
It could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the CAN 02 or ...

FH user could left the present source code here intact

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
SharedDataLength*SharedDataLength will be set to FH_CAN02_SharedDataBufLength_16Bits
SharedData*SharedData will be set to FH_CAN02_SharedDataBuf_16Bits
Returns
FH_ErrorInfo is returned
Note
FH user could use FH_CAN02_SharedDataBuf_16Bits in the receive ISR (Interrupt Service Routine) of the CAN 02 in a separate c file or anywhere else interested
For this to happen, FH user shall just include FH_CAN02_GlobalVariables.h there
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${CAN02}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_CAN}    Command=${CAN_Commands_GSD16}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  CAN02 => It indicates the CAN 02
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_CAN => It is the function in the message frame
  CAN_Commands_GSD16 => It is the command of the function in the message frame

Returns

In other word, FH_CAN02_ReceiveBuf array shall receive every byte of the received data there with FH_CAN02_ReceiveBufIndex as index of the array
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${CAN02}    05    00    00    0B    B8
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_CAN}    Command=${CAN_Commands_ReceiveW}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  CAN02 => It indicates the CAN 02
  05 => The expected number of bytes of data to be received through the <b>ISR</b> (Interrupt Service Routine) receive buffer (FH_CAN02_ReceiveBuf) of the <b>CAN 02</b>
  00    00    0B    B8 => Example data (0x00000BB8 mili-seconds) as timeout of receiving the expected number of bytes
                          FH_GlobalTimeCounter which is a global variable is utilized to calculate the timeout
                          FH_GlobalTimeCounter shall be incremented every <b>1 mili-second</b> in an interested timer <b>ISR</b> (Interrupt Service Routine) by <b>FH</b> user
                          For this to happen, <b>FH_GlobalTimerCount.h</b> shall be included in the interested timer <b>ISR</b> (Interrupt Service Routine) file
                          If the expected number of bytes are received, the function does not wait anymore for the remaining time of the timeout
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_CAN => It is the function in the message frame
  CAN_Commands_ReceiveW => It is the command of the function in the message frame
Warning
To be able to receive data through the CAN 02 of the micro-controller on which FH source code is ported, the FH_CAN_Init_CAN02 function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_CAN_ReceiveW_CAN02 function from RobotFramework, FH_CAN_Init_CAN02 function shall be invoked in order to initialize the CAN

FH_ErrorInfo FH_CAN_SSD32_CAN02 ( FH_RFCommunication_Message * fh_RFCommunication_Message)

This function sets the shared data buffer (FH_CAN02_SharedDataBuf_32Bits) of the CAN 02
The CAN 02 could be any interested CAN of the micro-controller on which FH source code is ported
FH_CAN02_SharedDataBuf_32Bits is a shared global 8 bit array which is a data bridge between RobotFramework and FH user code
FH_CAN02_SharedDataBuf_32Bits is optional
It could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the CAN 02 or ...

FH user could left the present source code here intact

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
Returns
FH_ErrorInfo is returned
Note
FH user could use FH_CAN02_SharedDataBuf_32Bits in the receive ISR (Interrupt Service Routine) of the CAN 02 in a separate c file or anywhere else interested
For this to happen, FH user shall just include FH_CAN02_GlobalVariables.h there
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${CAN02}    02    AA    BB    CC    DD    01    02    03    04
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_CAN}    Command=${CAN_Commands_SSD32}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  CAN02 => It indicates the CAN 02
  02 => Example interested length of the 32 bit data to be set
  AA    BB    CC    DD    01    02    03    04 => Example data (0xAABBCCDD and 0x01020304) to be set in <b>FH_CAN02_SharedDataBuf_32Bits</b>
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_CAN => It is the function in the message frame
  CAN_Commands_SSD32 => It is the command of the function in the message frame
