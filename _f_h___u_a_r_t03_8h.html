FH_ErrorInfo FH_UART_GSD16_UART03 ( FH_RFCommunication_Message * fh_RFCommunication_Message,
uint8_t * SharedDataLength,
uint8_t * SharedData )

This function gets the shared data buffer (FH_UART03_SharedDataBuf_16Bits) of the UART 03
The UART 03 could be any interested UART of the micro-controller on which FH source code is ported
FH_UART03_SharedDataBuf_16Bits is a shared global 16 bit array which is a data bridge between RobotFramework and FH user code
FH_UART03_SharedDataBuf_16Bits is optional
It could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the UART 03 or ...

FH user could left the present source code here intact

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
SharedDataLength*SharedDataLength will be set to FH_UART03_SharedDataBufLength_16Bits
SharedData*SharedData will be set to FH_UART03_SharedDataBuf_16Bits
Returns
FH_ErrorInfo is returned
Note
FH user could use FH_UART03_SharedDataBuf_16Bits in the receive ISR (Interrupt Service Routine) of the UART 03 in a separate c file or anywhere else interested
For this to happen, FH user shall just include FH_UART03_GlobalVariables.h there
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${UART03}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_UART}    Command=${UART_Commands_GSD16}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  UART03 => It indicates the UART 03
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_UART => It is the function in the message frame
  UART_Commands_GSD16 => It is the command of the function in the message frame

FH user could left the present source code here intact.

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
DataSize*DataSize will be set to the ISR (Interrupt Service Routine) receive buffer index (FH_UART03_ReceiveBufIndex) of the UART 03
Data*Data will be set to the ISR (Interrupt Service Routine) receive buffer (FH_UART03_ReceiveBuf) of the UART 03
Returns
FH_ErrorInfo is returned
Note
FH user shall use FH_UART03_ReceiveBuf and FH_UART03_ReceiveBufIndex in the receive ISR (Interrupt Service Routine) of the UART 03 in a separate c file
For this to happen, FH user shall just include FH_UART03_GlobalVariables.h there
In other word, FH_UART03_ReceiveBuf array shall receive every byte of the received data there with FH_UART03_ReceiveBufIndex as index of the array
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${UART03}    05    00    00    0B    B8
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_UART}    Command=${UART_Commands_ReceiveW}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  UART03 => It indicates the UART 03
  05 => The expected number of bytes of data to be received through the <b>ISR</b> (Interrupt Service Routine) receive buffer (FH_UART03_ReceiveBuf) of the <b>UART 03</b>
  00    00    0B    B8 => Example data (0x00000BB8 mili-seconds) as timeout of receiving the expected number of bytes
                          FH_GlobalTimeCounter which is a global variable is utilized to calculate the timeout
                          FH_GlobalTimeCounter shall be incremented every <b>1 mili-second</b> in an interested timer <b>ISR</b> (Interrupt Service Routine) by <b>FH</b> user
                          For this to happen, <b>FH_GlobalTimerCount.h</b> shall be included in the interested timer <b>ISR</b> (Interrupt Service Routine) file
                          If the expected number of bytes are received, the function does not wait anymore for the remaining time of the timeout
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_UART => It is the function in the message frame
  UART_Commands_ReceiveW => It is the command of the function in the message frame
Warning
To be able to receive data through the UART 03 of the micro-controller on which FH source code is ported, the FH_UART_Init_UART03 function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_UART_ReceiveW_UART03 function from RobotFramework, FH_UART_Init_UART03 function shall be invoked in order to initialize the UART
Note
The only solution for resetting the receive buffer index (FH_UART03_ReceiveBufIndex) is using FH_UART_ResetRB_UART03 function
In other words, FH_UART03_ReceiveBufIndex may have been already increased if any data has been already received
So it is normally required to invoke FH_UART_ResetRB_UART03 through RobotFramework to reset the receive buffer index (FH_UART03_ReceiveBufIndex) before invoking FH_UART_ReceiveW_UART03

FH_ErrorInfo FH_UART_SSD32_UART03 ( FH_RFCommunication_Message * fh_RFCommunication_Message)

This function sets the shared data buffer (FH_UART03_SharedDataBuf_32Bits) of the UART 03
The UART 03 could be any interested UART of the micro-controller on which FH source code is ported
FH_UART03_SharedDataBuf_32Bits is a shared global 8 bit array which is a data bridge between RobotFramework and FH user code
FH_UART03_SharedDataBuf_32Bits is optional
It could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the UART 03 or ...

FH user could left the present source code here intact

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
Returns
FH_ErrorInfo is returned
Note
FH user could use FH_UART03_SharedDataBuf_32Bits in the receive ISR (Interrupt Service Routine) of the UART 03 in a separate c file or anywhere else interested
For this to happen, FH user shall just include FH_UART03_GlobalVariables.h there

  ${Result}    SendMessage    &{Message}

  Comments:

  UART03 => It indicates the UART 03
  02 => Example interested length of the 32 bit data to be set
  AA    BB    CC    DD    01    02    03    04 => Example data (0xAABBCCDD and 0x01020304) to be set in <b>FH_UART03_SharedDataBuf_32Bits</b>
