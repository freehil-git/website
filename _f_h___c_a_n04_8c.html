Detailed Description

File Map:
FH_Functions, Section FH_CAN (CAN), Peripheral FH_CAN04 (CAN 04)
(The CAN04 (CAN 04) could be any interested CAN of the micro-controller on which FH source code is ported)

Note
To have a clean code, FH user should have the implementation of the initialization and send functions of the CAN 04 in a separate c file
Then FH user should include the header file (.h) of the related source file (.c) here
This header file shall include the following items:
The declaration of the initialization function of the CAN 04
The declaration of the send function of the CAN 04

FH user shall use FH_CAN04_ReceiveBuf and FH_CAN04_ReceiveBufIndex in the receive ISR (Interrupt Service Routine) of the CAN 04 in a separate c file
For this to happen, FH user shall just include FH_CAN04_GlobalVariables.h there
In other word, FH_CAN04_ReceiveBuf array shall receive every byte of the received data there with FH_CAN04_ReceiveBufIndex as index of the array

FH_CAN04_SharedDataBuf_8Bits, FH_CAN04_SharedDataBuf_16Bits and FH_CAN04_SharedDataBuf_32Bits are optional and could be left intact
They could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the CAN 04 , ...
They are controllable with some commands (described in this file) through RobotFramework

FH_ErrorInfo FH_CAN_SSD16_CAN04 ( FH_RFCommunication_Message * fh_RFCommunication_Message)

This function sets the shared data buffer (FH_CAN04_SharedDataBuf_16Bits) of the CAN 04
The CAN 04 could be any interested CAN of the micro-controller on which FH source code is ported
FH_CAN04_SharedDataBuf_16Bits is a shared global 8 bit array which is a data bridge between RobotFramework and FH user code
FH_CAN04_SharedDataBuf_16Bits is optional
It could be used by FH user to set some special data like receive filters on ISR (Interrupt Service Routine) of the CAN 04 or ...

FH user could left the present source code here intact

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework
Returns
FH_ErrorInfo is returned
Note
FH user could use FH_CAN04_SharedDataBuf_16Bits in the receive ISR (Interrupt Service Routine) of the CAN 04 in a separate c file or anywhere else interested
For this to happen, FH user shall just include FH_CAN04_GlobalVariables.h there
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${CAN04}    02    AA    BB    CC    DD
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_CAN}    Command=${CAN_Commands_SSD16}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  CAN04 => It indicates the CAN 04
  02 => Example interested length of the 16 bit data to be set
  AA    BB    CC    DD => Example data (0xAABB and 0xCCDD) to be set in <b>FH_CAN04_SharedDataBuf_16Bits</b>
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_CAN => It is the function in the message frame
  CAN_Commands_SSD16 => It is the command of the function in the message frame
