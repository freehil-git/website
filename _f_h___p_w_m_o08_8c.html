FH_ErrorInfo FH_PWMO_Init_PWMO08 ( FH_RFCommunication_Message * fh_RFCommunication_Message)

This function initializes the PWM Output 08
The PWM Output 08 could be any interested PWM Output of the micro-controller on which FH source code is ported
FH user should call the initialization function of the interested PWM Output here.

Parameters
None
Returns
FH_ErrorInfo is returned
Note
To have a clean code, just include the related header file
Then just call the function here
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${PWMO08}
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_PWMO}    Command=${PWMO_Commands_Init}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  PWMO08 => It indicates the PWM Output 08
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_PWMO => It is the function in the message frame
  PWMO_Commands_Init => It is the command of the function in the message frame
Warning
To be able to set or reset the PWM Output 08 of the micro-controller on which FH source code is ported, this function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_PWMO_SetStatus_PWMO08 function from RobotFramework, this function shall be invoked in order to initialize the PWM Output

FH_ErrorInfo FH_PWMO_SetStatus_PWMO08 ( FH_RFCommunication_Message * fh_RFCommunication_Message)

This function sets the PWM Output 08
The PWM Output 08 could be any interested PWM Output of the micro-controller on which FH source code is ported
FH user should call the set function of the interested PWM Output here.

Parameters
fh_RFCommunication_MessageThe message frame set by RobotFramework

FH_ErrorInfo is returned
Note
To have a clean code, just include the related header file
Then just call the function here
  ==============================================================================
                        ##### RobotFramework Example #####
  ==============================================================================
  @{MessageData} =    Create List    ${PWMO08}    00    00    00    F0    32
  &{Message}     =    Create Dictionary    DeviceAddress=${DeviceAdress_01}    Function=${Function_PWMO}    Command=${PWMO_Commands_SetStatus}    Data=@{MessageData}
  ${Result}    SendMessage    &{Message}

  Comments:

  PWMO08 => It indicates the PWM Output 08
  00    00    00    F0 => Example data as PWM frequency (0x000000F0) to be set through the <b>PWM Output 08</b>\n
  32 => Example data as PWM duty cycle (0x32 => as 50% duty cycle) to be set through the <b>PWM Output 08</b>\n
  DeviceAdress_01 => It shall be equal to the address of the device: FH_RFCommunication_DeviceAddress
  Function_PWMO => It is the function in the message frame
  PWMO_Commands_SetStatus => It is the command of the function in the message frame
Warning
To be able to set the PWM Output 08 of the micro-controller on which FH source code is ported, the FH_PWMO_Init_PWMO08 function shall be invoked once by RobotFramework firstly
In other words, before invoking FH_PWMO_SetStatus_PWMO08 function from RobotFramework, FH_PWMO_Init_PWMO08 function shall be invoked in order to initialize the PWM Output
