HIL (Hardware In the Loop) simulators are inseparable parts of developing Embedded System / Hardware / Software products
HIL simulators are utilized in different types of both Product Verification and Product Validation
HIL simulators could be utilized in Test Automation along with a test automation frame work like RobotFramework

Note
FH package is an open source software package (RobotFramework code + Embedded Software code) developed by Vahid Hasirchi
It simply helps Large, Medium and Small size corporations to quickly build their own interested yet powerful HIL simulators (How much quick? It depends, but maybe even less than 1 day or even less than 1 hour! in many cases)
FH package could be utilized in different types of both Product Verification and Product Validation in Embedded System / Hardware / Software products
The result is providing the corporations with the capabilities like:
1 - Running RobotFramework test cases in both Product Verification and Product Validation concepts
2 - Developing several production line's functional testers which are controllable and monitorable from just one centerelized computer in Product Validation concept
3 - Running Hardware labaritory tests in both Product Verification and Product Validation concepts
Attention
FH package is composed of 2 main parts as following:
1 - FH RobotFramework source code
2 - FH Embedded Software source code
And an interface provides the communication between FH RobotFramework source code in computer side and FH Embedded Software source code in HIL simulator side
The most simple interface is a simple USB to Serial interface which could connect the computer to one card (with a unique predefined address) of the HIL simulator
RS485 interface could be utilized which could connect the computer to several cards (with unique predefined addresses) of the HIL simulator
Attention
There are 3 main categories in FH source code as following:
1 - FH_Functions
2 - FH_Root
3 - FH_Setup
